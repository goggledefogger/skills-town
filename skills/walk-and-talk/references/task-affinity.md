# Task affinity — what's walk-and-talk-able vs desk-only

Not everything should be done on a walk, and the dividing line isn't difficulty — it's **GUI-bound vs scriptable**.

- **Walk-and-talk-able** — anything that is, at bottom, a terminal command or a conversation: deciding and talking through an approach, capturing decisions, moving / renaming / downloading files, transferring info or config, editing notes, git. You drive the Mac's session by voice from the phone; the Mac does the typing.
- **Desk-only** — anything that needs mouse-on-canvas / a GUI: a slicer (orient, supports, plate layout, slice settings), a CAD or image editor, anything you click and drag. The terminal can't reach it.

The good flow is therefore **split, not all-or-nothing**: talk it out and stage the scriptable parts on the walk, then sit down for the few minutes of GUI clicking. Example (3D printing): on the walk — choose the model, talk through the styling, download and file the model, write the project note; at the desk — import to the slicer, orient, support, slice, send to the printer.

**When unsure, ask "is this a command or a click?"** Commands go on the walk; clicks wait for the desk. Tell the user which parts you're staging vs which you're parking for the laptop, so nothing silently stalls.

## Cadence (tune this live)

Active mode has a dial between *just talking ideas out* and *nudging real work along through tasks and tools*. The right setting is the user's, learned over time, not fixed:

- Default to talking and capturing; escalate to "make progress on X now" only when they push.
- Keep audible updates sparse — headline plus an offer of depth, not a play-by-play of every tool call.
- **Wait for the user's start.** Don't barrel into the next step; finish a small unit, confirm, and let them say go.

## Say it vs type it

Voice is for the cheap and forgiving; the keyboard is for the precise and the secret.

- **Say it (voice):** ideas, decisions, short drafts, "do X", "skip", "more detail".
- **Type it (keyboard):** secrets — never dictate a token or password — plus exact code/commands, long precise text, anything a transcription typo would ruin.

The assistant runs the scripts; **the user never says script names.** They speak intent ("set up Telegram", "read me the brief") and the skill maps it to the command.
